#### 存储过程

存储过程（Stored Procedure）是在大型数据库系统中, 一组为了完成特定功能的 SQL 语句集, 存储在数据库中, 经过第一次编译后再次调用不需要再次编译, 用户通过指定存储过程的名字并给出参数（如果该存储过程带有参数）来执行它。存储过程是数据库中的一个重要对象。使用存储过程可以简化客户端的开发和维护，而且还可以提高客户端程序的运行性能。


#### 使用存储过程的优势

1. 提高性能

    因为存储过程是预编译的，在首次运行一个存储过程时，查询优化器对其进行分析、优化，并给出最终被存在系统表中的存储计划。而批处理的T-SQL语句每次运行都需要预编译和优化，所以速度就要慢一些。可以减少编译SQL语句的时间。

2. 降低了网络开销

    经过预编译的存储过程会进入缓存，所以对于要经常执行的存储过程，除了第一次外，其他次数的速度会明显提高。存储过程调用时只需用提供存储过程名和必要的参数信息，从而可减轻了网络的流量,降低了网络负载

3. 安全性高

    通过对用户授权对存储过程的访问权限，他们可以提供对特定数据的访问；可以控制数据的来源，防止 SQL 注入。


#### 使用存储过程的劣势

1. 如果业务逻辑太复杂，处理的数据量和计算量大的话会增加数据库服务器的压力。从而影响了数据库服务器对其他请求的响应。

2. 可移植性差，因为不同数据库的厂商提供的SQL函数是不一样的。

3. 如果需求经常变动，使用存储过程会使的数据库维护工作变得很难。所以针对于业务逻辑变动的比较复杂的程序不建议使用存储过程，而是将逻辑处理交给应用处理。

#### 简单的存储过程

1. 创建不带参数的储存过程

        -- 查询学生总数
        CREATE PROCEDURE PROC_SELECT_STUDENTS_COUNT
        AS
        	BEGIN
        		SELECT COUNT(Sno) FROM Student
        	END
        GO

        EXEC PROC_SELECT_STUDENTS_COUNT
        GO

2. 带有参数的储存过程

        -- 根据参数查询某个性别的总人数
        CREATE PROCEDURE PROC_SELECT_STUDENTS_BY_SEX_COUNT
        	@sex CHAR(2)
        AS
        	BEGIN
        		SELECT COUNT(Sno) FROM Student WHERE Ssex = @sex
        	END
        GO

        EXEC PROC_SELECT_STUDENTS_BY_SEX_COUNT N'男'
        EXEC PROC_SELECT_STUDENTS_BY_SEX_COUNT N'女'

        -- 查询姓氏为李的学生信息，带通配符
        CREATE PROCEDURE PROC_SELECT_STUDENT_BY_SURNNAME
            @surnname NVARCHAR(16) = '李%'  --默认值
        AS
        	BEGGIN
        		SELECT Sno, Sname, Sage, Sdept,Ssex FROM Student WHERE Sname LIKE @surnname
        	END
        GO

        EXEC PROC_SELECT_STUDENT_BY_SURNNAME
        EXEC PROC_SELECT_STUDENT_BY_SURNNAME N'钱%'
        EXEC PROC_SELECT_STUDENT_BY_SURNNAME N'王%'


3. 带有输出参数, 返回值

        -- 根据姓名查询学生的信息，返回学生的专业及年龄
        CREATE PROCEDURE PROC_SELECT_STUDENT_BY_NAME
            @name NVARCHAR(50),  --输入参数
            @sdept NVARCHAR(20) OUT,  --输出参数
            @age INT OUTPUT  --输入输出参数
        AS
        	BEGIN
        		SELECT @sdept = Sdept, @age = Sage FROM Student WHERE Sname = @name AND Sage = @age
        	END
        GO

        DECLARE @name NVARCHAR(16),
        			   @sdept NVARCHAR(8),
        			   @age INT;
        SET @name = N'李勇';
        SET @age = 19;
        EXEC PROC_SELECT_STUDENT_BY_NAME @name, @sdept OUT, @age OUTPUT;
        SELECT @sdept, @age;

4. 加密的存储过程

        -- 执行存储过程后，无法查看脚本或者导出脚本
        CREATE PROCEDURE PROC_SELECT_STUDENT_WITH_ENCRYPTION
            WITH ENCRYPTION  --加密
        AS
        	BEGIN
        		SELECT * FROM Student
        	END
        GO

        EXEC PROC_SELECT_STUDENT_WITH_ENCRYPTION

#### 使用存储过程进行增删改操作

1. 向表中插入记录

        CREATE PROCEDURE PROC_INSERT_STUDENT
        	@Sno INT,
        	@Sname VARCHAR(10),
        	@Ssex CHAR(2),
        	@Sage INT,
        	@Sdept VARCHAR(16)
        AS
        	BEGIN
        		INSERT INTO Student(Sno, Sname, Ssex, Sage, Sdept) VALUES(@Sno, @Sname, @Ssex, @Sage, @Sdept);
        	END
        GO

        EXEC PROC_INSERT_STUDENT 1523531, N'张三', '男', 25, '自动化专业'

2. 根据传入的参数,修改表中的记录

        -- 根据传入的id，修改学生的信息
        CREATE PROCEDURE PROC_UPDATE_STUDENT
        	@Sno INT,
        	@Sname VARCHAR(10),
        	@Ssex CHAR(2),
        	@Sage INT,
        	@Sdept VARCHAR(16)
        AS
        	BEGIN
        		UPDATE Student SET Sname = @Sname, Ssex = @Ssex, Sage = @Sage, Sdept = @Sdept WHERE Sno = @Sno
        	END
        GO

        EXEC PROC_UPDATE_STUDENT 1523531, N'李四', '男', 25, '自动化专业'


3. 创建存储过程删除学生记录

        -- 根据传入的ID删除分数表相应的记录
        CREATE PROCEDURE PROC_DELETE_SCORE
        	@Sno INT
        AS
        	BEGIN
        		DELETE FROM Score WHERE Sno = @Sno
        	END
        GO

        EXEC PROC_DELETE_SCORE 9512102

4. 存储过程实现分页查询

        -- 使用row_number函数分页
        CREATE PROCEDURE PROC_SELECT_BY_PAGE
        	@startindex int,
        	@endindex int
        AS
        	BEGIN
        		SELECT * FROM
        				(SELECT Sno, Sname, Ssex, Sage, Sdept, ROW_NUMBER() OVER(ORDER BY Sno ASC) AS rowNumber FROM Student) AS temp
        				WHERE TEMP.rowNumber BETWEEN @startindex AND @endindex
        	END
        GO

        SELECT * FROM Student
        EXEC PROC_SELECT_BY_PAGE 3, 6
